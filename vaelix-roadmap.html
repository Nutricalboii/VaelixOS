Before We Start
Known Flaws to Fix in v2.0

These are the exact issues inherited from v1.1. Every phase below directly addresses one or more of these.

⚠
Live session boot failure — kernel /boot/vmlinuz missing on some USB/HDD enclosures. ISO not universal yet.
⚠
Hardware gaps — acpi_osi suspend/resume, Nvidia persistence, hybrid graphics not fully handled.
⚠
Onboarding wizard incomplete — visual first-run bridge exists but lacks full guided flow.
⚠
No Kali tooling — only available via Flatpak optionally. No sandboxed pentest layer built in.
⚠
Security hardening is surface-level — AppArmor active but no hardened kernel, no immutable paths, no TPM integration.
⚠
VHI edge cases — Lite/Velocity modes untested on exotic or very low-end hardware.
⚠
No Rust tooling layer — vx commands are Shell/Python; no Rust binaries for speed and memory safety.
System Design
v2.0 Architecture Blueprint

Five distinct layers, each independently improvable. Vaelix Renaissance is modular by design.

🖥
UI / Compositor Layer
Wayland-first KDE Plasma 6. NixOS-inspired spatial layout. Fluid KWin animations with GPU compositing. Glassmorphic Amethyst Noir identity.
Wayland KDE Plasma 6 KWin SDDM
🔒
Security Hardening Layer
GrapheneOS-inspired desktop hardening. Hardened kernel patches, AppArmor profiles, sandboxed Flatpak, TPM-backed FDE, minimal attack surface.
hardened-linux AppArmor Firejail TPM2
🦀
Core Tooling in Rust
Rewrite speed-critical vx commands as Rust binaries. Memory safe, fast, zero-dep executables. Shipped as part of the base layer.
rustc cargo clippy vx-core
🕵️
Kali Sandbox Layer
Optional but installable pentest toolkit in an isolated namespace. nmap, gobuster, hydra, sqlmap, wireshark — sandboxed, no host system risk.
nmap gobuster sqlmap Podman
⚡
VHI 2.0 — Adaptive Intel
Improved Universal Hardware Intelligence. Auto-detects GPU vendor, RAM tier, storage type. Applies Lite / Standard / Velocity profile at first boot.
VHI thermald ZRAM BFQ/none
🧭
Onboarding & Identity
Full first-run wizard. Atmosphere picker, VHI report, hardware summary, dev/security mode toggle. Vaelix identity on boot splash, login, terminal.
Plymouth SDDM Theme VCC Wizard Starship
Execution Plan
Phase-by-Phase Roadmap

Each phase is self-contained. Paste one phase at a time into your AI IDE — Google's Gemini or any agent will understand the task blocks.

Phase 1 — Wayland Migration & UI Renaissance
UI LAYER
Migrate from X11 to full Wayland. Upgrade to KDE Plasma 6. Rebuild all animations and glassmorphic theming from scratch to match NixOS spatial quality but with Vaelix Amethyst Noir identity. Fix the VCC Atmosphere tab layout bug seen in screenshots.
Install KDE Plasma 6 + Wayland compositor: plasma-workspace-wayland, kwin-wayland
Set SDDM to launch Plasma Wayland session by default in /etc/sddm.conf.d/vaelix.conf
Enable KWin effects: Wobbly Windows OFF, Magic Lamp minimize ON, Glide ON, translucency ON
Create custom KWin script for Vaelix window open/close animations (fade + slight scale from 0.95)
Update Vaelix Atmosphere QML panel — fix the "choose your theme" card layout overflow visible in LinkedIn screenshot
Add "Auto Scheduler" theme switcher: sunrise = Titanium Velocity, sunset = Amethyst Noir, midnight = Executive Noir
Rebuild VCC sidebar in PyQt6 to be Wayland-compatible (no X11 window positioning hacks)
Set accent color to respond to power mode: amethyst on AC, cyan-dim on battery
Verify KRunner, Dolphin, all GTK apps use native Wayland (set WAYLAND_DISPLAY and QT_QPA_PLATFORM=wayland)
Test suspend/resume on Wayland — fix black screen resume bug
Phase 2 — Security Hardening (GrapheneOS-Inspired)
SECURITY LAYER
Apply desktop-grade hardening inspired by GrapheneOS principles. Hardened kernel, locked-down services, app sandboxing, TPM-backed disk encryption, minimal network attack surface.
Install linux-hardened kernel or apply kernel lockdown mode: security=apparmor lsm=apparmor,lockdown
Enable TPM2-backed LUKS2 encryption: clevis + tpm2-tools for auto-unlock on trusted hardware
Write AppArmor profiles for: Firefox, VCC, Dolphin, terminal emulator
Harden /etc/sysctl.conf: disable IPv6 if unused, restrict ptrace, enable ASLR full, disable kernel pointers in logs
Set up Firejail defaults for untrusted apps (install firejail + firetools)
Enforce Flatpak permission model: revoke filesystem=host by default, use flatpak override per app
Create vx security audit command: checks AppArmor status, LUKS status, open ports, failed login attempts
Add VCC Security tab: shows active profiles, TPM status, last audit result, one-click re-enforce button
Disable Bluetooth + WiFi on fresh boot (user enables via VCC toggle — explicit consent model)
Log all sudo usage to /var/log/vaelix/sudo.log with timestamp + command
Phase 3 — Rust Core & vx 2.0
RUST LAYER
Rewrite speed-critical vx commands as compiled Rust binaries. Add Rust + full dev stack to the default install. This is the "clueless young person building an OS in Rust" moment.
Install Rust toolchain via official installer (not distro package): curl https://sh.rustup.rs | sh
Create vx-core Rust project: cargo workspace with sub-crates for each vx command
Port vx doctor to Rust: reads SMART data, battery health, service status — faster than Python version
Port vx clean to Rust: dry-run preview mode, then clean with confirmation prompt
Write vx bench in Rust: CPU, RAM, disk speed benchmark with Vaelix-styled ASCII output
Keep vx Shell wrapper as entrypoint — Rust binaries are called under the hood transparently
Add vx dev init command: scaffolds a dev workspace with git, venv, cargo, Docker in one shot
Pre-install: rustc, cargo, cargo-audit, clippy, Node.js LTS, Python 3.12 + pip, Docker, lazygit
Integrate cargo audit into vx doctor for Rust project vulnerability scanning
Add Starship prompt Rust segment: shows active Rust version and cargo project name in terminal
Phase 4 — Kali Sandbox & Pentest Layer
HACKER LAYER
Integrate optional Kali tooling in an isolated Podman/namespace container — not system-level. User activates via vx pentest on. No host risk, full tool access.
Install Podman (rootless container runtime) as the isolation layer for Kali tools
Build vaelix-kali Podman image: based on kalilinux/kali-rolling, with nmap, gobuster, hydra, sqlmap, nikto, wireshark-cli
Write vx pentest on: pulls/starts the kali container and drops user into an isolated shell
Write vx pentest off: stops and removes the running container cleanly
Add VCC "Dev Tools" tab: pentest toggle, container status, list of available tools
Create vx scan [target] wrapper: runs nmap + basic recon inside the container with output piped to host terminal
Ensure WiFi adapter is passed through to container for wireless scanning if needed (--device flag)
Store scan results in ~/vaelix-pentest-logs/ on host — container has no persistent write access elsewhere
Add disclaimer modal in VCC when enabling pentest mode: "only use on networks you own or have permission to test"
Phase 5 — VHI 2.0, Onboarding Wizard & ISO Polish
RELEASE LAYER
Fix the live session boot bug. Complete the onboarding wizard. Stress-test VHI Lite/Velocity on edge hardware. Build the final ISO with all layers integrated. Ship v2.0.
Fix live ISO boot: ensure vmlinuz is correctly symlinked in /boot for casper live session
Test ISO on VirtualBox, QEMU/KVM, bare metal USB boot — verify all three work before release
Upgrade VHI: add GPU vendor detection (nvidia/amd/intel), apply correct driver and power profile per vendor
VHI Lite mode: disable Baloo indexing, reduce animation scale to 0.5x, cap background daemons
VHI Velocity mode: enable full animations, XanMod kernel params, gaming mode auto-prep
Build complete onboarding wizard (PyQt6): Step 1 = Atmosphere pick, Step 2 = VHI report, Step 3 = dev/security mode, Step 4 = done + VCC launch
Add Nvidia persistence fix: write /etc/systemd/system/nvidia-persistenced.service override
Add acpi_osi fix for suspend/resume: write kernel parameter to /etc/default/grub and run update-grub
Rebuild Plymouth boot splash: animated Vaelix logo with amethyst particle effect
Run full release audit: vx doctor, security audit, snapshot test, ISO boot test — target Trust Score 97/100
Tag release as v2.0-renaissance on GitHub, update README with new architecture diagram
